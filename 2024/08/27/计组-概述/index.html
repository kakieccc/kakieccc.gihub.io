计算机组成原理——概述

2024-08-27

计算机发展历程

计算机硬件一共经历了四次迭代变化：

  1. 第一代计算机：电子管时代，逻辑元件采用电子管，使用机器语言进行编程，体积大、容量小、成本高、速度慢
  2. 第二代计算机：晶体管时代，逻辑元件采用晶体管，高级语言和编译程序出现
  3. 第三代计算机：中小规模集成电路时代，逻辑元件采用中小规模集成电路；半导体存储器开始取代磁芯存储器；开始有了分时操作系统
  4. 第四代计算机：超大规模集成电路时代，逻辑元件采用大规模和超大规模集成电路；产生了微处理器（CPU）；Cache，并行，流水线，虚拟存储器等概念出现并运用

计算机系统层次结构

计算机系统的组成

计算机 = 软件 + 硬件

计算机性能的好坏取决于软硬件功能的总和

冯诺依曼计算机的特点

  • 采用“存储程序”的工作方式：将事先编制好的程序和数据送入主存再执行。一旦程序开始执行就不需要操作人员干预，可以自动逐条执行指令
  • 计算机由运算器、存储器、控制器、输入设备、输出设备五大部件组成
  • 指令和数据以同等地位存放于存储器内；均以二进制数表示，形式上没有区别，但计算机应该能加以区分
  • 指令由操作码和地址码组成，操作码指出操作类型，地址码指出操作数的地址
  • 指令在存储器内按顺序存放
  • 机器以运算器为中心，IO设备和存储器之间的数据传送必须先经过运算器

当代计算机的特点

当代计算机建立在冯诺依曼计算机的基础上，将五大件的中心从运算器改为了存储器。

现代计算机可以认为由三大部分组成：

  • CPU：运算器和控制器
  • IO设备：输入设备和输出设备
  • 主存储器

其中CPU和主存储器合起来又可称为主机，IO设备又可称为外部设备。

计算机的功能部件

  1. 输入设备

  2. 输出设备

  3. 存储器：用于存放数据和程序

    分为主存储器（内存/主存）和辅助存储器（外存储器/外存）

    CPU能直接访问的存储器是主存储器，辅助存储器中的信息必须调入主存储器后才能被CPU所访问

    主存储器中包含了存储体、各种逻辑部件及控制电路等：

    • 存储体：由许多存储单元组成，每个存储单元又包含若干个存储元件，每个存储元件都可以存一个0或1。那么一个存储单元可存储一串二进制代码，这串二进制代码为一个存储字，其位数称为存储字长

    • MAR：存储器地址寄存器，存放欲访问的存储单元的地址

      MAR的位数对应了存储单元的最大个数，例如MAR为10位，则有2^10=1024个存储单元

    • MDR：存储器数据寄存器，用来存放从存储单元中取出的代码或者准备向某存储单元写入的代码

      通常情况下MDR位数与字长相同

    • ❗MAR和MDR虽然是存储器的一部分，但是在现代计算机中却是存在于CPU中，此外，Cache也存在于CPU中

  4. 运算器：完成算术运算和逻辑运算，并将运算的中间结果暂存在运算器内

    运算器最少包括三个寄存器（ACC累加器，MQ乘商寄存器，X操作数寄存器）和一个ALU（算术逻辑单元）

    运算器内还有程序状态寄存器（PSW），也叫标志寄存器，用来寄存ALU运算得到的标志信息（结果是否溢出、有无进位、借位、结果是否为负等等）

  5. 控制器：控制、指挥程序和数据的输入、运行以及处理运算结果。

    控制器由PC程序计数器、IR指令寄存器和CU控制单元组成。

    完成一条指令需要三步：取指令（PC）、分析指令（IR）、执行指令（CU）

    • PC：存放当前欲执行指令的地址，具有计数和自动加“1”的功能，可以自动形成下一条指令的地址
    • IR：存放当前的指令，IR的内容来自MDR，指令的操作码被送至CU，地址码送至MAR用于取操作数
    • CU：分析当前指令所需完成的操作并发出微操作命令序列

系统软件和应用软件

  • 系统软件：保证计算机系统运行的基础软件，通常作为系统资源提供给用户使用。主要包括操作系统（OS）、数据库管理系统（DBMS）、语言处理程序、分布式软件系统等

  • 应用软件：用户为解决某个应用领域中的各类问题而编制的程序。

三个级别的语言和程序

三种语言

  • 机器语言：也称二进制代码语言，是计算机唯一可以直接识别和执行的语言

  • 汇编语言：用助记符代替二进制的指令代码。汇编语言需要先经过汇编程序进行翻译成机器语言才能执行

  • 高级语言：为方便程序设计人员编写的程序。高级语言需要经过编译程序编译成汇编语言程序，然后经过汇编程序进行翻译成机器语言才能执行；也可以直接由高级语言程序翻译成机器语言程序。

三种程序

翻译程序分类：

  • 汇编程序（汇编器）：将汇编语言程序翻译成机器语言程序
  • 解释程序（解释器）：将源程序中的语句按执行顺序逐条翻译成机器指令并执行
  • 编译程序（编译器）：将高级语言程序翻译成汇编语言或机器语言程序

❗编译器是将高级语言编写的源程序一次性全部翻译成机器语言程序，解释器是将源程序的一条语句翻译成机器语言并执行，每次执行语句都要翻译。一般来讲解释型语言的效率要更低

计算机系统的层次结构

软件和硬件之间的界面就是指令集体系结构（ISA），ISA定义了一台计算机可执行的所有指令的集合，以及每条指令的作用是什么、每条指令的用法是什么

计算机系统的工作原理

从源程序到可执行文件

以C++为例。

  1. 预处理阶段：预处理器对源程序hello.c进行处理，处理为hello.i
  2. 编译阶段：编译器对于处理后的源程序hello.i进行编译，生成汇编语言源程序hello.s
  3. 汇编阶段：汇编器将生成的汇编语言源程序hello.s翻译成机器指令，生成可重定位目标文件hello.o
  4. 链接阶段：链接器将多个可重定位目标文件.o和标准库函数合并为一个可执行文件

“存储程序”思想原理

  1. 程序执行前先把第一条指令的地址存到PC中
  2. 将PC中的内容作为地址访问主存储器
  3. 指令执行过程中PC执行自动加“1”形成下一条指令地址
  4. 当前指令执行完后，根据PC的值就可以找到下一条指令从而继续执行程序

完整的指令执行过程

下面我们按照完整的取指令、分析指令、执行指令的过程来走一遍取数指令执行过程

  • 取指令：1-4步。
    • 首先PC取指令（地址）并送入到MAR中：PC -> MAR
    • 存储器读MAR内容（地址），在存储体中按地址找到对应内容
    • 存储体找到对应内容后送到MDR
    • MDR将指令的具体内容送给IR：MDR -> IR
  • 分析指令：第5步。
    • IR将指令的操作码送到CU进行分析：OP(IR) -> CU，CU分析完指令后，知道这是一条取数指令
  • 执行指令：第6-9步。
    • IR中的地址码送到MAR：Ad(IR) -> MAR
    • 存储体接收MAR中的地址并在其内部找到对应内容
    • 存储体将查到的内容送到MDR中
    • 最后MDR将取出来的数传给ACC：MDR -> ACC

计算机组成和计算机体系结构

  • 计算机体系结构：计算机体系结构是程序员眼中能见到的的计算机系统的属性和功能特性。例如指令集、数据类型、寻址技术等。只要两个计算机指令集、数据类型、寻址类型等相等，就说这两个计算机体系结构相同

  • 计算机组成：计算机组成是体系结构的具体实现形式。是指如何实现计算机体系结构所体现的属性

例如，计算机是否有乘法指令是体系结构设计的问题；而这条指令是由特定的乘法单元实现，还是通过重复使用系统的加法单元来实现，则是组成的问题

计算机硬件的主要技术指标

数据通路带宽

数据总线一次所能并行传送信息的位数。

存储容量

存储容量包括主存容量和辅存容量

主存容量指主存中存放二进制代码的总位数。存储容量 = 存储单元个数 * 存储字长

字长

  • 机器字长：指CPU一次能处理数据的位数，简称字长。通常来说机器字长与CPU中寄存器位数相同。字长越长数的表示范围越大，精度越高，性能越高
  • 存储字长：一个存储单元存储的二进制位数。即MDR的位数。一般情况下等于机器字长。
  • 指令字长：一个指令字中包含的二进制位数。可以等于机器字长，也可以是机器字长的整数倍。如果指令字长等于存储字长，那么取指周期等于机器周期

周期

  • 时钟周期：CPU中最小的时间单位，每个动作都至少需要一个时钟周期
  • 机器周期：也叫CPU周期。为了便于管理，我们会把一条指令的执行划分成多个阶段（取指、译码、执行等），每一阶段完成一个基本操作。完成一个基本操作的时间称为机器周期。一般规定机器周期为一次访存所需的时间，由若干时钟周期组成。
  • 指令周期：CPU执行一条机器指令所需的时间，由若干机器周期组成。

运算速度

吞吐量和响应时间

  • 吞吐量：系统在单位时间内处理请求的数量

  • 响应时间：用户向计算机发送一个请求到系统对该请求做出响应并获得结果的等待时间

CPU主频（时钟频率）：CPU每秒能执行的时钟周期数，数值上等于时钟周期的倒数

CPI：执行一条指令所需的时钟周期数。执行一条指令的耗时 = CPI * 时钟周期

CPU执行时间：运行一个程序所花费的时间。CPU执行时间 = CPU时钟周期数 * 时钟周期 = CPU时钟周期数 / 主频 = （指令条数 * CPI）/主频

IPS：每秒执行多少条指令。IPS = 指令条数 / CPU执行时间 = 主频 / CPI

FLOPS：每秒执行多少次浮点运算

❗IPS和FLOPS前可以加修饰来表示数量级：

  • K：千 10^3
  • M：百万级 10^6
  • G：十亿 10^ 9
  • T：万亿 10^12
  • P：千万亿 10^15
  • E：百京 10^18
  • Z：十万京 10^21

基准程序：专门用来进行性能评价的一 组程序。

几个专业术语

  1. 系列机：具有基本相同体系结构，使用基本相同指令系统的多个不同型号计算机组成的一个系列产品
  2. 兼容：不同型号的计算机系统中的软件或硬件的通用性
  3. 固件：将程序固化在ROM中组成的部件
